ESP32 Wheel Setup

DIY Bluetooth Steering Wheel

Build a low-latency, wireless racing controller using an ESP32 microcontroller and an MPU6050 gyroscope sensor. Perfect for games like Assetto Corsa, F1, and Euro Truck Simulator.

How It Works (Architecture)

1. Physical Input

You mount the MPU6050 behind a steering wheel. As you rotate the wheel, gravity shifts relative to the sensor.

2. ESP32 Processing

The ESP32 reads raw I2C data, applies mathematical filters to remove hand jitter, and calculates the exact angle.

3. BLE HID Output

The angle is transmitted over Bluetooth. Your PC sees it as a standard plug-and-play Gamepad joystick. No drivers required.

Bill of Materials

  • ESP32 Development Board

    Ensure it has built-in Bluetooth (most do). A NodeMCU ESP-WROOM-32 is perfect.

  • MPU-6050 Module

    A cheap 6-axis IMU (Gyro + Accelerometer). Often comes with pre-soldered headers.

  • 4x Jumper Wires

    Female-to-Female wires to connect the ESP32 pins to the sensor pins directly.

  • Power Source

    A Micro-USB/USB-C cable connected to your PC, or a small USB power bank mounted to the wheel.

Sensor Physics & Math Simulator

Why do we need complex code? Because physics is messy. Play with the wheel below to see how raw sensor data is filtered into usable game input.

High α = Smoother but slower. Low α = Fast but jittery.

2. Real-Time Signal Processing (Complementary Filter)

Accel (Noisy) Gyro (Drifts) Filtered Output
The Math inside the ESP32:
Angle = (α * (Angle + Gyro_Rate * Time_Step)) + ((1 - α) * Accel_Angle)
This combines the short-term speed of the Gyroscope with the long-term stability of the Accelerometer (Gravity).

steering_wheel.ino 
#include <BleGamepad.h>
#include <Wire.h>
#include <Adafruit_MPU6050.h>
#include <Adafruit_Sensor.h>

BleGamepad bleGamepad("ESP32 Wheel", "DIY Setup", 100);
Adafruit_MPU6050 mpu;

// Filter Variables
float angle = 0;
float alpha = 0.96; 
unsigned long lastTime = 0;

void setup() {
  Serial.begin(115200);
  
  // Initialize BLE Gamepad
  bleGamepad.begin();
  
  // Initialize MPU6050
  if (!mpu.begin()) {
    Serial.println("Failed to find MPU6050");
    while (1) delay(10);
  }
  
  // Configure MPU ranges
  mpu.setAccelerometerRange(MPU6050_RANGE_2_G);
  mpu.setGyroRange(MPU6050_RANGE_500_DEG);
  mpu.setFilterBandwidth(MPU6050_BAND_21_HZ);
  
  lastTime = millis();
}

void loop() {
  if (bleGamepad.isConnected()) {
    sensors_event_t a, g, temp;
    mpu.getEvent(&a, &g, &temp);

    // Calculate time elapsed
    unsigned long currentTime = millis();
    float dt = (currentTime - lastTime) / 1000.0;
    lastTime = currentTime;

    // 1. Accelerometer Angle (Roll)
    // atan2 gives radians, multiply by 57.2958 to get degrees
    float accelAngle = atan2(a.acceleration.y, a.acceleration.z) * 57.2958;

    // 2. Gyro Rate (X-axis rotation speed)
    float gyroRate = g.gyro.x * 57.2958;

    // 3. Complementary Filter
    angle = alpha * (angle + gyroRate * dt) + (1.0 - alpha) * accelAngle;

    // 4. Map Angle (-90 to +90) to Gamepad Axis (-32767 to 32767)
    // Constrain to prevent overflow if turned past 90 degrees
    float constrainedAngle = constrain(angle, -90, 90);
    int mappedAxis = map(constrainedAngle, -90, 90, -32767, 32767);

    // 5. Send to PC (X axis, Y, Z, rZ, rX, rY)
    bleGamepad.setAxes(mappedAxis, 0, 0, 0, 0, 0);
    
    // Optional: Add button logic here
    // if(digitalRead(BUTTON_PIN) == LOW) bleGamepad.press(BUTTON_1);
  }
  
  delay(10); // Run loop at ~100Hz
}

Calibration & Deadzones

When you hold a wheel straight, your hands naturally tremble slightly. If we send this raw tremble to the game, the car will drive erratically on straightaways. We fix this by coding a Deadzone—a range near zero where input is ignored.
